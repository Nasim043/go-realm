# Go Pointers

পয়েন্টার হলো এমন একটি ভ্যারিয়েবল যা অন্য একটি ভ্যারিয়েবলের **মেমোরি অ্যাড্রেস** ধরে রাখে। সহজ ভাষায়: পয়েন্টার ভ্যালু নিজে না রেখে ভ্যালুটি **কোথায় আছে** সেটা জানে।

> 🏠 **অ্যানালজি:** ভ্যারিয়েবল হলো বাড়ি, পয়েন্টার হলো সেই বাড়ির ঠিকানা.

---

## ১. Core Operators 📌

| Operator | নাম | কাজ | উদাহরণ |
|:---:|:---|:---|:---|
| `&` | Address-of | ভ্যারিয়েবলের মেমোরি অ্যাড্রেস বের করে | `ptr := &x` |
| `*` | Dereference | পয়েন্টারের অ্যাড্রেসে থাকা ভ্যালু পড়ে/পরিবর্তন করে | `fmt.Println(*ptr)` |
| `*T` | Pointer type | `T` টাইপের পয়েন্টার ডিক্লেয়ার করে | `var p *int` |

```go
x := 42
ptr := &x           // ptr = x-এর মেমোরি অ্যাড্রেস (যেমন 0xc000014070)

fmt.Println(x)      // 42       — সরাসরি ভ্যালু
fmt.Println(&x)     // 0xc...   — x-এর অ্যাড্রেস
fmt.Println(ptr)    // 0xc...   — ptr-এ থাকা অ্যাড্রেস (same)
fmt.Println(*ptr)   // 42       — ptr-এর অ্যাড্রেসে যা আছে

*ptr = 100          // x-এর ভ্যালু পরিবর্তন করা হলো
fmt.Println(x)      // 100 ✅
```

## ২. কেন পয়েন্টার দরকার?

| কারণ | ব্যাখ্যা |
|:---|:---|
| **ফাংশনের ভেতর থেকে original পরিবর্তন** | Go-তে সব argument copy হিসেবে যায়; pointer দিলে original পরিবর্তন সম্ভব |
| **বড় struct-এর copy এড়ানো** | Pointer পাস করলে পুরো struct copy হয় না — performance ভালো |
| **Optional value প্রকাশ করা** | `nil` দিয়ে "কোনো ভ্যালু নেই" বোঝানো যায় |

```go
// ❌ Copy হয় — original পরিবর্তন হয় না
func doubleWrong(n int) {
    n = n * 2
}

// ✅ Pointer — original পরিবর্তন হয়
func double(n *int) {
    *n = *n * 2
}

func main() {
    x := 5
    doubleWrong(x)
    fmt.Println(x) // 5 — অপরিবর্তিত

    double(&x)
    fmt.Println(x) // 10 ✅
}
```

## ৩. Pointer এবং Struct (Production Usage)

```go
type User struct {
    Name  string
    Email string
}

// ❌ Value — struct-এর copy পাস হয়
func greetValue(u User) {
    u.Name = "Changed" // original-এ কোনো প্রভাব নেই
}

// ✅ Pointer — original struct পাস হয়
func greetPointer(u *User) {
    u.Name = "Changed" // original পরিবর্তন হয়
}

func main() {
    u := User{Name: "Rahim", Email: "rahim@email.com"}

    greetValue(u)
    fmt.Println(u.Name) // "Rahim" — অপরিবর্তিত

    greetPointer(&u)
    fmt.Println(u.Name) // "Changed" ✅
}
```

> **💡 Note:** Struct field access-এ Go স্বয়ংক্রিয়ভাবে pointer dereference করে। তাই `(*u).Name` না লিখে সরাসরি `u.Name` লেখা যায়।

## ৪. nil Pointer — সবচেয়ে Common Bug ⚠️

Pointer declare করলে কিন্তু initialize না করলে এর default value হয় `nil`.

```go
var ptr *int        // nil — কোনো অ্যাড্রেস নেই
fmt.Println(ptr)    // <nil>
fmt.Println(*ptr)   // 💥 panic: nil pointer dereference
```

**সমাধান — সবসময় nil check করুন:**

```go
func printValue(ptr *int) {
    if ptr == nil {
        fmt.Println("কোনো ভ্যালু নেই")
        return
    }
    fmt.Println(*ptr)
}
```

**`new()` দিয়ে initialized pointer তৈরি:**

```go
ptr := new(int)     // *int তৈরি, zero value (0) দিয়ে initialize
*ptr = 42
fmt.Println(*ptr)   // 42
```

## ৫. Value vs Pointer — কোনটা কখন? 🔥

| পরিস্থিতি | ব্যবহার করুন |
|:---|:---:|
| ছোট ডেটা, শুধু read করবেন | Value |
| ফাংশনে original পরিবর্তন করতে হবে | Pointer |
| বড় struct পাস করছেন | Pointer |
| "ভ্যালু নেই" বোঝাতে হবে (optional) | Pointer (`nil`) |
| `sync.Mutex` বা non-copyable type | Pointer (বাধ্যতামূলক) |
| Slice, Map, Channel পাস করছেন | Value (এগুলো নিজেই reference type) |

## ৬. Go সবসময় Pass by Value (Most Asked Interview Topic) 🔥

> **এটি Go-র সবচেয়ে বেশি জিজ্ঞেস করা interview প্রশ্নগুলোর একটি।**

Go-তে ফাংশনে argument পাস করার সময় **সবসময় একটি copy তৈরি হয়**। Pointer পাস করলেও copy হয় — তবে সেক্ষেত্রে **address-এর copy** হয়।

```go
func changeValue(n int) {
    n = 99 // int-এর copy, original অপরিবর্তিত
}

func changePointer(n *int) {
    *n = 99 // address-এর copy, কিন্তু সেই address-এ গিয়ে original পরিবর্তন করা হয়
}

func main() {
    x := 10

    changeValue(x)
    fmt.Println(x) // 10 — অপরিবর্তিত

    changePointer(&x)
    fmt.Println(x) // 99 ✅ — original পরিবর্তিত
}
```

| কী পাস হয় | কপি হয় | Original পরিবর্তন হয়? |
|:---|:---|:---:|
| `int`, `string`, `struct` (value) | পুরো ভ্যালুর কপি | ❌ না |
| `*int`, `*struct` (pointer) | address-এর কপি (৪/৮ বাইট) | ✅ হ্যাঁ (dereference করলে) |

> 💡 **মূল কথা:** Pointer পাস করা মানে "pass by reference" নয়। Go-তে reference নেই। Pointer হলো address-এর value — সেটাও copy হিসেবেই যায়।

## ৭. Pointer vs Reference Type (Interview Critical)

Go-তে কিছু built-in type আছে যেগুলো **ইতিমধ্যে reference**-এর মতো আচরণ করে:

| Type | Pointer দরকার? | কারণ |
|:---:|:---:|:---|
| `int`, `string`, `bool`, `struct` | ✅ দরকার হলে | Value type — copy হয় |
| `slice` | ❌ সাধারণত না | Internal pointer আছে |
| `map` | ❌ না | Reference type |
| `channel` | ❌ না | Reference type |
| `interface` | ❌ না | Reference type |

```go
// Map-এ pointer দরকার নেই — ফাংশনে পাস করলেও original update হয়
func addUser(m map[string]int, name string) {
    m[name] = 1 // ✅ original map-এ update হয়
}
```

## ৮. Common Mistakes ⚠️

| ভুল | সমস্যা | সমাধান |
|:---|:---|:---|
| nil pointer dereference | runtime panic | ব্যবহারের আগে `if ptr != nil` চেক করুন |
| Loop variable pointer | সব pointer একই শেষ ভ্যালু দেখায় | loop-এর ভেতরে আলাদা ভ্যারিয়েবল তৈরি করুন |
| Slice/Map-এ অকারণে pointer | unnecessary complexity | এরা নিজেই reference, pointer লাগে না |
| `new()` ও `&{}` confusion | — | দুটোই কাজ করে, `&Struct{}` বেশি idiomatic |

```go
// ❌ Classic loop pointer bug
ptrs := make([]*int, 3)
for i := 0; i < 3; i++ {
    ptrs[i] = &i // সব pointer শেষে i=3 দেখাবে!
}

// ✅ সমাধান — local copy তৈরি করুন
for i := 0; i < 3; i++ {
    v := i
    ptrs[i] = &v
}
```

## ৯. Interview Cheat Sheet

**Q: পয়েন্টার কী এবং কেন ব্যবহার করা হয়?**
> পয়েন্টার একটি ভ্যারিয়েবলের মেমোরি অ্যাড্রেস ধরে রাখে। ব্যবহার হয়: original value পরিবর্তন করতে, বড় struct copy এড়াতে এবং optional value প্রকাশ করতে।

**Q: `&` এবং `*`-এর পার্থক্য কী?**
> `&` দিয়ে ভ্যারিয়েবলের অ্যাড্রেস নেওয়া হয়। `*` দিয়ে পয়েন্টারের অ্যাড্রেসে থাকা ভ্যালু access করা হয় (dereference)।

**Q: Pointer-এর zero value কী?**
> `nil` — মানে এটি কোনো মেমোরি অ্যাড্রেস point করছে না।

**Q: Slice বা Map-এ কি Pointer দরকার?**
> সাধারণত না। Slice ও Map নিজেই internally reference type — ফাংশনে পাস করলে original-ই কাজ হয়।

**Q: `new(T)` এবং `&T{}`-এর পার্থক্য?** 📌
> দুটোই `*T` return করে। `new(T)` zero value দিয়ে initialize করে। `&T{field: value}` দিয়ে নির্দিষ্ট ভ্যালু দেওয়া যায় — production-এ `&T{}` বেশি ব্যবহৃত।

**Q: Go-তে কি Pass by Reference আছে?**
> নেই। Go সবসময় Pass by Value। Pointer পাস করলেও address-এর একটি copy যায় — reference যায় না।

**Q: nil pointer dereference কীভাবে এড়াবেন?**
> Pointer ব্যবহারের আগে সবসময় `if ptr != nil` চেক করুন। অথবা `new()` বা `&T{}` দিয়ে initialize করে নিন।
