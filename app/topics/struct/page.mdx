# Go Structs

Go ল্যাঙ্গুয়েজে ডেটা মডেলিং এবং অর্গানাইজ করার মেরুদণ্ড হলো স্ট্রাক্ট।
## ১. স্ট্রাক্ট কী?

স্ট্রাক্ট হলো একটি কাস্টম ডেটা টাইপ, যা বিভিন্ন ধরণের ডেটাকে একটি ভ্যারিয়েবলের বা প্যাকের ভেতর রাখে। একে আপনি ডেটাবেসের টেবিল বা অন্যান্য ল্যাঙ্গুয়েজের ক্লাস (Class)-এর সাথে তুলনা করতে পারেন (যদিও Go-তে ক্লাস নেই)।

 A struct is a collection of members of different data types into a single variable.

## ২. স্ট্রাক্ট তৈরি ও ব্যবহার

```go
package main
import "fmt"

type Person struct {
  Name   string
  Age    int
  Salary int
}

func main() {
  // ১. সাধারণ নিয়ম
  var p1 Person
  p1.Name = "Rahim"
  p1.Age = 30
  
  // ২. struct literal (সবচেয়ে বেশি ব্যবহৃত)
  p2 := Person{
      Name: "Karim",
      Age: 25,
      Salary: 40000,
  }

  fmt.Println(p1, p2)
}

```

## ৩. Exported vs Unexported (Visibility) 🔥

Go ডেভেলপমেন্টে এটি বোঝা সবচেয়ে জরুরি। Go-তে `public` বা `private` কিওয়ার্ড নেই। অক্ষরের কেস (Case) দিয়ে এটি নিয়ন্ত্রণ করা হয়।

* **বড় হাতের অক্ষর (Uppercase):** যদি ফিল্ডের নামের প্রথম অক্ষর বড় হাতের হয় (যেমন `Name`), তবে এটি **Exported (Public)**। অন্য প্যাকেজ থেকে এটি এক্সেস করা যাবে।
* **ছোট হাতের অক্ষর (Lowercase):** যদি প্রথম অক্ষর ছোট হাতের হয় (যেমন `name`), তবে এটি **Unexported (Private)**। শুধুমাত্র নিজের প্যাকেজের ভেতর এটি দেখা যাবে।

```go
type User struct {
    Username string // Public: অন্য প্যাকেজ থেকে দেখা যাবে
    password string // Private: বাইরের প্যাকেজ থেকে দেখা যাবে না (Security)
}

```

## ৪. কনস্ট্রাক্টর প্যাটার্ন (Constructor Pattern) 🔥

অন্য ল্যাঙ্গুয়েজে `constructor` থাকে, কিন্তু Go-তে নেই। ইন্ডাস্ট্রিতে আমরা সাধারণত একটি ফাংশন বানিয়ে এই কাজ করি, যার নাম সাধারণত `New` দিয়ে শুরু হয়। এটি একটি **Best Practice**।

```go
type Employee struct {
    Name string
    Role string
}

// এটি কনস্ট্রাক্টরের মতো কাজ করছে
func NewEmployee(name string) *Employee {
    return &Employee{
        Name: name,
        Role: "Junior Developer", // ডিফল্ট ভ্যালু সেট করা যায়
    }
}

func main() {
    // অবজেক্ট তৈরির স্ট্যান্ডার্ড উপায়
    emp := NewEmployee("Siam")
    fmt.Println(emp)
}

```

## ৫. নেস্টেড স্ট্রাক্ট (Nested Structs) 
একটি স্ট্রাক্টের ফিল্ড হিসেবে অন্য একটি স্ট্রাক্ট ব্যবহার করাকে Nested Struct বলে। রিয়েল লাইফ প্রজেক্টে এটি খুব বেশি ব্যবহৃত হয়।

```go
type Address struct {
    City, Country string
}

type Person struct {
    Name    string
    Address Address // Nested Struct
}

func main() {
    p := Person{
        Name: "Alice",
        Address: Address{
            City: "Dhaka",
            Country: "Bangladesh",
        },
    }
    // এক্সেস করা: ডট অপারেটর চেইন করে ব্যবহার করতে হয়
    fmt.Println(p.Address.City) // Output: Dhaka 
}

```

## ৬. স্ট্রাক্ট মেথড ও রিসিভার (Methods & Receivers) 📌

Go-তে ক্লাসের বদলে স্ট্রাক্টের সাথে ফাংশন বা মেথড যুক্ত করা হয় । মেথড ডিফাইন করার সময় **রিসিভার** ঠিক করা খুব গুরুত্বপূর্ণ।

* **Value Receiver `(p Person)`**: এটি স্ট্রাক্টের একটি কপি নিয়ে কাজ করে। মেথডের ভেতরে কোনো পরিবর্তন করলে আসল স্ট্রাক্টে প্রভাব পড়ে না । 


* **Pointer Receiver `(p *Person)`**: এটি আসল স্ট্রাক্টটি নিয়ে কাজ করে। এখানে ফিল্ডের মান পরিবর্তন করলে আসল স্ট্রাক্টেও পরিবর্তন হয় ।



```go
type Rectangle struct {
    width, height float64
}

// Value Receiver: শুধুমাত্র হিসাব করার জন্য
func (r Rectangle) Area() float64 {
    return r.width * r.height
}

// Pointer Receiver: স্ট্রাক্টের ডেটা পরিবর্তন করার জন্য
func (r *Rectangle) Scale(factor float64) {
    r.width = r.width * factor   // মেইন স্ট্রাক্টের মান পরিবর্তন হবে
    r.height = r.height * factor
}

```

## ৭. স্ট্রাক্ট এম্বেডিং (Inheritance-এর বিকল্প) 

Go-তে ইনহেরিটেন্স নেই, তবে **Composition** বা এম্বেডিং ব্যবহার করে আমরা কোড রি-ইউজ করতে পারি। একটি স্ট্রাক্টকে অন্য স্ট্রাক্টের ভেতর সরাসরি ঢুকিয়ে দেওয়া যায় ।

```go
type Author struct {
    FirstName string
    LastName  string
}

type Book struct {
    Title  string
    Author // Embedding: এখানে কোনো ফিল্ড নাম দেওয়া হয়নি
}

func main() {
    b := Book{
        Title: "Go Programming",
        Author: Author{FirstName: "John", LastName: "Doe"},
    }
    
    // Promoted Field: সরাসরি প্যারেন্ট ফিল্ড এক্সেস করা যায়!
    fmt.Println(b.FirstName) // b.Author.FirstName লেখার দরকার নেই
}

```

## ৮. অ্যানোনিমাস স্ট্রাক্ট (Anonymous Structs) 

মাঝে মাঝে আপনার এমন একটি স্ট্রাক্ট দরকার হতে পারে যা আপনি শুধু একবারই ব্যবহার করবেন। সেক্ষেত্রে আলাদা টাইপ তৈরি না করে **অ্যানোনিমাস স্ট্রাক্ট** ব্যবহার করা হয় । এটি দ্রুত এবং টেম্পোরারি কাজের জন্য খুব ভালো ।

```go
// টাইপ ডিফাইন ছাড়াই সরাসরি স্ট্রাক্ট তৈরি
data := struct {
    Title string
    Code  int
}{
    Title: "Success",
    Code:  200,
}

```

## ৯. JSON ট্যাগস (Web Dev-এর জন্য জরুরি)

API তৈরির সময় স্ট্রাক্ট ফিল্ড এবং JSON কি (Key)-এর নাম আলাদা হতে পারে। এজন্য **Tags** ব্যবহার করা হয়।

```go
type Product struct {
    ID    int     `json:"product_id"` // JSON এ "product_id" দেখাবে
    Name  string  `json:"name,omitempty"` // ভ্যালু না থাকলে ফিল্ড হাইড করবে
    Price float64 `json:"-"` // এই ফিল্ডটি কখনোই JSON এ যাবে না (Security)
}

```

## ১০. মনে রাখার মতো টিপস (Best Practices)

1. **তুলনা (Equality):** স্ট্রাক্টের সব ফিল্ড যদি `comparable` (যেমন int, string) হয়, তবেই `==` দিয়ে তুলনা করা যায়। Slice বা Map থাকলে `==` কাজ করবে না। 


2. **Nil Pointer:** স্ট্রাক্ট পয়েন্টার যদি `nil` হয়, তবে তার ফিল্ড এক্সেস করলে কোড ক্র্যাশ করবে। তাই পয়েন্টার রিসিভ করার আগে চেক করে নিন। 
```go
type Book struct {
  title string
}

func main() {
  var b *Book // এটি এখন nil
  
  if b != nil {
      fmt.Println(b.title)
  } else {
      fmt.Println("Book is nil")
  }
}

```


3. **Memory:** বড় স্ট্রাক্ট ফাংশনে পাস করার সময় `Pointer` ব্যবহার করুন, এতে মেমোরি বাঁচে (পুরো স্ট্রাক্ট কপি হয় না)।

## ১১. OOP → Go স্ট্রাক্ট চিটশিট 📋

অন্যান্য OOP ল্যাঙ্গুয়েজ জানলে নিচের টেবিলটি Go-এর স্ট্রাক্ট দ্রুত বুঝতে সাহায্য করবে।

| # | OOP Concept | Go-তে সমতুল্য | মূল বিবরণ |
|:---:|:---|:---|:---|
| 1 | `class` | `type Foo struct{}` | কাস্টম ডেটা টাইপ তৈরির একমাত্র উপায় |
| 2 | `public` field | Uppercase field — `Name string` | যেকোনো প্যাকেজ থেকে সরাসরি অ্যাক্সেসযোগ্য |
| 3 | `private` field | Lowercase field — `name string` | শুধুমাত্র একই প্যাকেজে দৃশ্যমান, বাইরে থেকে লুকানো |
| 4 | `constructor` | `NewFoo()` factory function | `New` দিয়ে শুরু — ডিফল্ট ভ্যালু সেট করার Best Practice |
| 5 | `method` | Receiver function | `func (f Foo) Bar()` — স্ট্রাক্টের সাথে যুক্ত ফাংশন |
| 6 | `this` / `self` | Receiver variable | `r`, `p` ইত্যাদি — মেথডের ভেতরে নিজের স্ট্রাক্টের রেফারেন্স |
| 7 | Read-only method | Value Receiver `(f Foo)` | স্ট্রাক্টের কপিতে কাজ করে, মূল ডেটা অপরিবর্তিত থাকে |
| 8 | Mutating method | Pointer Receiver `(f *Foo)` | আসল স্ট্রাক্টের ডেটা পরিবর্তন করতে পারে |
| 9 | `inheritance` | Struct Embedding | ফিল্ড নাম ছাড়া embed → child সরাসরি parent-এর ফিল্ড পায় |
| 10 | `null` object | `nil` pointer — `var p *Foo` | nil pointer-এ ফিল্ড অ্যাক্সেস করলে runtime panic হয় |

> **💡 মনে রাখুন:** Go-তে `class`, `public`, `private`, `extends`, `this` — এই কিওয়ার্ডগুলো নেই। কিন্তু উপরের টেবিলের সব ধারণাই Go-তে ভিন্নভাবে বিদ্যমান।
