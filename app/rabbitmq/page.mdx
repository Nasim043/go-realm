## RabbitMQ কী?

RabbitMQ হলো একটি **message broker**। এটি producer থেকে message নিয়ে queue-তে রাখে, rule অনুযায়ী route করে, এবং consumer-কে deliver করে। এর ফলে service গুলো tightly coupled না হয়ে **asynchronous** এবং **resilient** হতে পারে।

![RabbitMQ Core Architecture](/rabbitmq/core-architecture.svg)

## কেন Go backend-এর সাথে RabbitMQ শেখা দরকার?

| বিষয় | কেন গুরুত্বপূর্ণ |
| --- | --- |
| Background jobs | Email, report generation, image processing queue-তে পাঠানো যায় |
| Decoupling | API service আর worker service আলাদা রাখা যায় |
| Reliability | retry, ack, dead-letter queue দিয়ে fault tolerance বাড়ে |
| Throughput | burst traffic absorb করা যায় |
| Go fit | goroutine + channels mindset থেকে RabbitMQ বুঝতে সুবিধা হয় |

## এই ট্র্যাক কার জন্য?

- যারা Go backend লিখছে এবং queue-based architecture বুঝতে চায়
- যারা pub/sub, async job processing, retry, DLQ production-grade ভাবে শিখতে চায়
- যারা শুধু tutorial না, **reasoning + implementation** দুটোই চায়

## শুরু করার আগে যা জানা ভালো

- Go basics: `struct`, `goroutine`, `context`, package layout
- Docker বা local service run করার বেসিক
- HTTP API এবং JSON payload সম্পর্কে ধারণা
